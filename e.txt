### Update /home_data endpoint
The phase tendencies shown on home screen should be personalised after enough data is collected. Logic:
- If user has fewer than 14 emotion_logs: use default phase tendencies
- If user has 14+ emotion_logs: query emotion_logs filtered by current phase, find top 3 most frequent emotions, use those as personalised tendencies with a note "Based on your patterns"
- Same logic for sleep: if sleep_logs exist for current phase, show their actual average hours vs phase average
- Same logic for diet: if diet_logs exist, show their most common cravings for current phase
- Pass a "personalised": true/false flag in the response so the frontend can show "Based on your data" vs "Typical for this phase"

### New endpoint: POST /rate_intervention (update existing)
Body: {intervention_id, intervention_name, intervention_type, rating, phase, cycle_day, duration_seconds}
- Save full log to intervention_logs collection in Firebase
- If rating >= 4, add to user's effective_interventions array
- If rating <= 2, note as ineffective (track separately as ineffective_interventions)
- Return {"status": "ok", "message": "Thanks for rating — Kira will remember this"}

### New endpoint: GET /personalised_intervention
Query params: ?emotion=anxiety&phase=luteal
- Fetches user's intervention_logs from Firebase
- Finds interventions rated >= 4 that match the emotion
- If found: returns those first (personalised recommendations)
- If not enough data: falls back to default matching by best_for array
- Returns ordered list of up to 3 recommended interventions with explanation: "Based on what's helped you before" vs "Popular for this phase"
